# 设计模式


## 工厂模式
* Why
    * 使用工厂模式或者工厂方法，我们只需要这个对象，而不需要了解这个对象的具体创建步骤
    * 我们可以交给一个工厂对象来帮我们生产产品对象，工厂最好也是抽象的类
    * 模块直接的耦合，最好都是面向接口和抽象类


## 迭代器模式
* Why
    * 我们需要一个对象内部的某一集合属性，迭代器防止了暴露对象内存属性细节给外部

## bridge模式

* Why
    * 解耦抽象类和它的实现

    ```
    When:
            A             
         /     \
        Aa      Ab
       / \     /  \
     Aa1 Aa2  Ab1 Ab2

     Refactor to
         A         N
      /     \     / \
    Aa(N) Ab(N)  1   2
    ```
